# Offerwall

**Offerwall** messages are designed to help you quickly experiment with new monetization opportunities to grow revenue on your site. Display an Offerwall message to your site visitors to offer them alternative ways to support your site and gain access to your content, such as by viewing a rewarded ad or paying a small fee.

For more information in official google documentation click <a href="https://support.google.com/admanager/answer/13860694?hl=en" target="_blank">**here**</a>

### Config example

```js filename="Offerwall config" {4, 12, 11} copy
<script>
	window.vpb=window.vpb||{};
	window.vpb.offerwall:{
                // adUnitPath: '/22639388115/rewarded_web_example',
                unitId: 58,
                logoUrl: 'https://tsn.ua/static/pub/img/logo.svg?v=988',
                timeOffsetHours: 24 ,
                scroll: 100,
                minPageViewsCount: 3,
                backdropShow: 1,
                // parentNetworkCode: '111',
                // childNetworkCode: '3333'
            },
</script>
```

Offerwall config has three additional prperties:

- `adUnitPath` - we can use this parameter instead of unitId , for example if target unit is not contained in our system 
- `parentNetworkCode`, `childNetworkCode` - use when multiple customer accounts  

Mandatory parameters
- `unitId` - unit id in the system
- `logoUrl` - path to the logo image
- `timeOffsetHours` - format repeat timer, timeOffsetHours: 24 - it works one time ufter 24 hours
- `scroll` - scroll value in pixels when banner will be rendered
- `minPageViewsCount` - number of transitions to site pages when banner will be rendered
- `backdropShow` - could be 1 or 0 

when `backdropShow:1` - show background:
