# Passback Anchors Description

**Passback Anchors** - extand functionality of `Anchors` and give us posibility to specify HTML or JavaScript that will be executed if no ads are returned from the primary demand sources. This helps to ensure that the ad space is still utilized effectively.
Click <Link href="/Description/anchors">**here**</Link> to read about the `Anchors`.

The functionality was developed as part of task <Link href="https://adtelligent.atlassian.net/browse/HBMP-421"  title="Click to open!">`HBMP-421`</Link> .

</StyledButton>

Passback config has one additional property:

- `passback` This is the key part of the configuration for handling unfilled ad requests.

  `html`: Contains the HTML and JavaScript for the passback ad. In this example, the passback code loads an ad from Ideal Media if the primary demand sources do not fill the ad slot.

When we adding string to JSON config we need to use special syntaxis like as escape special characters such as double quotes and backslashes in the HTML content to ensure it is valid JSON.For example The double quotes inside the HTML content are escaped with backslashes: `\"https:\/\/jsc.idealmedia.io\/i\/d\/idealmedia.io.1601438.js\"`.
